Top 5 Pro Tips for AI-Assisted Coding

I’ve been using AI to help me create coding projects for a couple of years now, and it’s been a learning process that started far from perfect and which has steadily improved with time and practice. In those two or so years of building projects with AI, I have managed to go from not knowing what the hell npm or pip means to regularly building projects with both. If you’re not familiar with these tools yet, that’s perfectly fine. In today’s article, we’re going to go over my top five tips for how to get the most out of AI when building coding projects. This article is meant for those without a background in traditional web development or programming.


#1: Interact with AI as if it were a freelancer you hired.

The reason this matters is because if you’d hired someone to help you make your idea a reality by writing code for it, you would not expect perfect results on the first pass. You would understand that the process involves two-way communication, compromise, and often multiple revisions before you end up with close to what you wanted. Working with AI is no different in this regard. It’s a collaboration, and treating it like one will honestly get you a lot further than expecting to write a one-sentence prompt and get perfect results.

#2: Accept that you will need to learn at least a few basic skills.

You won’t have to learn how to write any actual code, but you will need to become familiar with a few basic workflows that will allow you to run, install, test and ship the projects you build with AI. Yes, there is a bit of a learning curve. But the huge advantage you have with AI is that it can walk you through unfamiliar processes step-by-step, explain and help solve errors you will definitely encounter along they way, and ultimately troubleshoot just about anything. Your only real limit today is your own patience and capacity for navigating unfamiliar territory. 

#3: Think in terms of user journeys to describe what you want.

If you’re making a web application or a software product and don’t have actual coding skills, sometimes the best way to communicate the functionality that you want without knowing the terminology is to describe user journeys to AI instead. As an example, I might write something like: “I’d like to make a website where users can create an account, login, and create promotional campaigns for free ebook giveaways. Visitors to the campaign pages for a specific ebook should be able to enter their email address and then be redirected to the free download and also emailed with the download link.” AI will know how to translate that to backend and frontend code.

#4: Start basic and work your way up.

Some coding projects are bigger and more complicated or involved than others, and it’s important to define the scope early on. I have some things I’ve built with AI that took an afternoon and are still useful and serve a purpose, and other projects which I’ve devoted months to working on and improving. Some are still under development after several months. The important milestone to pass in order to ship or share the project with others is having a minimal viable product. This is the idea stripped down to its most core functions. Once you have achieved that, everything else is decoration or adding features and can be worked on down the road. 

#5: For complex coding projects involving a huge number of files, use a specialized AI.

For a long time, I used ChatGPT for my coding projects but the limitations became clear once I was trying to work on large projects over longer periods of time. For simple coding, sometimes showing ChatGPT the structure of the files can communicate enough to get meaningful help. However, you will likely have to individually upload the specific files he asks for after doing this. It’s much easier to use an AI tool like Cursor, which offers a desktop software you can download and lets you open your project folder so that Cursor can see, edit and have access to every actual file of code. This is far more powerful and sustainable. In fact, ChatGPT is great at writing stellar Cursor prompts.
